RANKX

Posted on 2021/08/31 | In DAX速查词典

概述

返回一个值，

迭代一参的表并计算公式后得到多个值的有序列表，然后在当前上下文中计算公式得到一个值，在有序列表中寻找该值的排名。

语法

参数 属性 描述
Table 返回表的表达式
Expression 在一参产生的行上下文和外部的上下文中计算的公式
Value 可选 在当前上下文下计算的公式，如果省略，则使用二参的Expression，通常会省略该参数
Order 可选 升序还是降序：0/FALSE/DESC表示降序；1/TRUE/ASC表示升序；默认是DESC
Ties 可选 名次是紧凑型还是跳跃型：SKIP表示跳跃型（1、1、3）；Dense表示紧凑型（1、1、2）；默认是SKIP

举例

第三参数在当前上下文中计算，没有时则使用第二参数

第三参数的使用场景

计算产品颜色在自定义的销售等级列表里面的排名

第四参数，升序或降序（默认降序）

第五参数，紧凑型或跳跃型（默认跳跃型）

FILTER

Posted on 2021/08/31 | In DAX速查词典

概述

返回一个被筛选的表。

作为迭代函数，迭代一参产生行上下文，常见用法有2种：

  • 对一参的表中的数据进行筛选
  • 利用行上下文转换，对数据动态筛选

语法

参数 属性 描述
Table 要筛选的表，可以是公式计算出的临时表
FilterExpression 在迭代的行上下文中计算的布尔表达式，结果返回TRUE/FALSE

举例

1，筛选出产品颜色是黑色的产品表
2，使用RELATED和RELATEDTABLE筛选
3，利用行上下文转换进行动态筛选
4，FILTER可以筛选变量表

ADDCOLUMNS

Posted on 2021/08/31 | In DAX速查词典

概述

返回一个添加了新列后的表。

我个人的使用场景是，当我使用FILTER筛选表时，我会把筛选公式作为ADDCOLUMNS的派生列公式，这样可以看到筛选公式的计算结果是否符合预期。

注意:在ADDCOLUMNS的派生列中引用其他列，该派生列不具备数据沿袭，换句话说，派生列直接无法当作筛选器。

语法

参数 属性 描述
Table 要筛选的表，可以是公式计算出的临时表
Name 可多个 派生列的列名
Expression 可多个 派生列的公式

举例

证明：派生列不具备数据沿袭

ALL

Posted on 2021/08/25 | In DAX速查词典

概述

作为表函数，返回表的所有行或返回列的所有去重值

作为CALCULATE调节器，清除扩展表或指定列上的筛选器，不返回任何值。

语法

参数 属性 描述
TableNameOrColumnName 可选 模型中的表或列
ColumnName 可选、多参 同一张基表中的列

注意

用作表函数时

  • 参数是表时，ALL(表)会返回表的所有行，不会对重复行去重。
  • 参数是列时，ALL(列)会返回列的所有去重值，参数是多列时，ALL( 列1 , 列2… )会返回所有列的非重复组合（相当于于excel中的选中多列后去重）

ALL会返回由参照完整性导致的空白行

用作调节器时（相当于REMOVEFILTERS）

  • 当ALL用做CALCULATE或CALCULATETABLE的筛选参数，且位于代码最外层时，ALL起到的是调节器作用，不返回表（），只清除指定表或列上的筛选器。

  • 当ALL不带参数时，会将模型上所有的筛选器都清除掉。

举例

ALL函数用作表函数
ALL函数用作CALCULATE调节器

ALLSELECTED

Posted on 2021/08/25 | In DAX速查词典

概述

激活参数列/表的筛选器堆栈（The Stack Of Filter Contexts）最上面的影子筛选器，忽略行上下文转的筛选器。

1，作为表函数
  • 参数是列时，只返回列上的直接筛选器，忽略交叉筛选
  • 参数是表时，相当于对每一列都加了ALLSELECTED，会激活每一列筛选器堆栈最上层的影子筛选器，返回被所有列上的筛选器共同筛选后的表（考虑交叉筛选）。
2，作为CALCULATE调节器

比较简单，不返回任何值，只是激活参数列/表的筛选器堆栈最上面的影子筛选器。不带参数时，相当于对所有表都加了个ALLSELECTED。

语法

参数 属性 描述
TableNameOrColumnName 可选 模型中的表或列
ColumnName 可选、可多个 同一张基表中的列

举例

1，ALLSELECTED用作表函数

1.1 当参数是列时

ALLSELECTED(列)，激活位于参数列筛选器堆栈的最上面的影子筛选器，返回直接筛选器的值，忽略交叉筛选和行上下文转换的筛选器。若列上没有显式筛选器/影子筛选器，则返回列的所有去重值。

1.1.1， 参数是列，筛选器堆栈最上面是影子筛选器时

解析：上图在姓名列的筛选器堆栈是：

筛选器类型 描述 值
影子筛选器 由ADDCOLUMNS迭代产生的影子筛选器 “t_” 张三、李四
显式筛选器 由最外层CALCULATETABLE添加的显式筛选器，位于筛选器堆栈最下面 张三

由于姓名列的筛选器堆栈最上层是影子筛选器，会被激活，优先级高于下面的显式筛选器，发生覆盖，所以姓名列最终的筛选器就是影子筛选器，值是张三、李四，所以ALLSELECTED(dim[姓名])会返回张三、李四。

1.1.2 参数是列，筛选器堆栈最上面是显式筛选器时

解析：上图在姓名列的筛选器堆栈是：

筛选器类型 描述 值
显式筛选器 ADDCULUMNS派生列公式中，CALCULATE添加的显式筛选器 王五
行上下文转换形成的筛选器 ADDCULUMNS派生列公式中，行上下文遇到CALCULATE发生转换 行上下文的值
影子筛选器 由ADDCOLUMNS迭代产生的影子筛选器 “t_” 张三、李四
显式筛选器 由最外层CALCULATETABLE添加的显式筛选器，位于筛选器堆栈最下面 张三

由于姓名列的筛选器堆栈最上层是显式筛选器，不是影子筛选器，ALLSELECTED无法激活位于中间层的影子筛选器（就算激活了也会被上面的显式筛选器覆盖掉），因此姓名列最终的筛选器是显式筛选器“王五”，所以ALLSELECTED(dim[姓名])会返回“王五”。

1.1.3 参数是列，证明ALLSELECTED忽略行上下文转换的筛选器

解析：上图在姓名列的筛选器堆栈是：

筛选器类型 描述 值
行上下文转换形成的筛选器 ADDCULUMNS派生列公式中，行上下文遇到CALCULATE发生转换 行上下文的值
影子筛选器 由ADDCOLUMNS迭代产生的影子筛选器 “t_” 张三、李四
显式筛选器 由最外层CALCULATETABLE添加的显式筛选器，位于筛选器堆栈最下面 张三

由于姓名列的筛选器堆栈最上层是行上下文转换形成的筛选器，ALLSELECTED会当它不存在，跳过它继续看下一层，发现是影子筛选器，于是影子筛选器被激活，姓名列最终的筛选器就是影子筛选器，所以ALLSELECTED(dim[姓名])会返回张三、李四。

1.1.4 参数是列，证明ALLSELECTED(列)不考虑交叉筛选

解析：上图在“姓名”列的筛选器堆栈是：

筛选器类型 描述 值
行上下文转换形成的筛选器 ADDCULUMNS派生列公式中，行上下文遇到CALCULATE发生转换 行上下文的值
影子筛选器显式筛选器 由ADDCOLUMNS迭代产生的影子筛选器 “t_” 张三、李四
显式筛选器 由最外层CALCULATETABLE添加的显式筛选器，位于筛选器堆栈最下面 张三

最内层在“别名”列也加了个筛选器，别名为“王五”

ALLSELECTED( dim[姓名] )忽略姓名列筛选器堆栈最上面的行上下文转换的筛选器，激活影子筛选器“张三”和“李四”，因此姓名列最终只有该影子筛选器，所以ALLSELECTED(dim[姓名])返回“张三”和李四，不受“别名列”筛选器的影响。

1.1.5 列上没有显式筛选器/影子筛选器，则返回列所有去重值

1.2 当参数是表时

相当于对每一列都加了ALLSELECTED，会激活每一列筛选器堆栈最上层的影子筛选器，返回被所有列上的筛选器共同筛选后的表，考虑交叉筛选。跟参数是列的最大区别是，参数是列时只考虑直接筛选器，不考虑交叉筛选。

1.2.1 证明：参数是表时考虑交叉筛选

解析：上图在“姓名”列的筛选器堆栈如下：

筛选器类型 描述 值

因此姓名列上的影子筛选器被激活并覆盖下面的显式筛选器，最终姓名列的筛选器是：“张三”和“李四”；而在“别名”列同样也有筛选器，是由派生列代码最外层的CALCULATE添加的筛选器，值是“王五”，两个筛选器共同筛选dim表得到一张空表。因此CONCATENATEX函数返回的结果为空。

将“别名”列上的筛选器值改为“李四”后，交叉筛选后，dim表被筛选为李四的数据。

  • 忽略行上下文转换的筛选器

影子筛选器 由ADDCOLUMNS迭代产生的影子筛选器 “t_” 张三、李四
显式筛选器 由最外层CALCULATETABLE添加的显式筛选器，位于筛选器堆栈最下面 王五

由于筛选器堆栈最上面是显式筛选器，不是影子筛选器，因此ALLSELECTED不执行任何操作，该显式筛选器会覆盖下面所有的筛选器，最终姓名列的筛选器值为“张三”，因此CONCATENATEX函数返回值为“张三”。

显式筛选器 由最外层CALCULATETABLE添加的显式筛选器，位于筛选器堆栈最下面 王五

CALCULATE

Posted on 2021/08/14 | In DAX速查词典

概述

修改第一参数计算时的筛选上下文，返回一个值。

所谓筛选上下文，其实就是被筛选器筛选后的模型可见数据。

语法

参数 属性 描述
Expression 要计算的表达式
Filter 可选、可多个 筛选器参数有2种：
1，表函数（支持布尔表达式，其本质是表函数的简化写法）
2，CALCULATE调节器
注意：筛选器参数计算时，互不影响。

CALCULATE计值流

点击查看CALCULATE视频讲解（时长40分钟）

CALCULATE计值流可分为3步

  • 第1步，计算显式筛选器，注意，显式筛选器是在CALCULATE代码外部环境计算的
  • 第2步，计算新模型，新模型计算分2步：先行上下文转换形成隐式筛选器，再应用CALCULATE调节器
  • 第3步，计算最终模型，用第一步计算的显式筛选器筛选第二步的新模型，得到最终模型可见数据（筛选上下文），第一参数在最终模型的可见数据下计算

注意，无论公式怎么写，度量值所处的外部环境都不会变，使用CALCULATE也只是临时修改了第一参数的计算环境而已。

关于筛选器之间的关系

  • 不同列上的筛选器会共同起作用筛选模型。
  • 同一列存在多个筛选器，内层筛选器优先级更高（显式筛选器>隐式筛选器>外部筛选器），优先级高的筛选器会覆盖掉优先级低的筛选器，可以理解为优先级低的筛选器不再起作用，相当于被清除了。

举例

1，筛选参数是布尔表达式时的完整版写法

布尔表达式本质上是表函数，因此是CALCULATE的显式筛选器，内层的显式筛选器会覆盖外层筛选器

1.1，使用“=”的布尔表达式，判断是否等于单个值

对于布尔表达式的写法，CALCULATE的显式筛选器是一个值为2007年的表，位于日期表的”年”这一列，而度量值外部在该列也存在筛选器，但会被内部的显示筛选器覆盖掉，因此每个单元格都是计算的2007年的销量。

1.2，使用“IN”的布尔表达式，用于判断多个值

2，使用CALCULATE调节器KEEPFILTERS使内部和外部筛选器取“交集”，筛选模型

已知同一列上的内层显式筛选器会覆盖外部的筛选器，使用KEEPFILTERS（内部筛选器），使内部筛选器不再覆盖外部筛选器，2者会取“交集”筛选模型。

3，行上下文转换形成的筛选器 覆盖 外层筛选器

行上下文转换形成的筛选器也叫做“隐式筛选器”，位于中层，会覆盖掉同一列上的外层的筛选器

4，计值流实战

可以看到，”计值流实战”的结果始终是交货量的3倍，其计值流如下：

当迭代2007时，我们严格按照CALCULATE计值流分析：

  • 1，计算显示筛选器

    这里的显式筛选器就是VALUES( ‘dim_日期表’[年] )，其计算环境是在CALCULATE代码外部环境计算，因此返回值为”2007”。

  • 2，计算新模型（上下文转换+调节器调节）

    这里的行上下文是2007，行上下文转换变成隐式筛选器，而外层同一列已有筛选器，因此外层筛选器被覆盖（失效），模型只有隐式筛选器2007，

    然后调节器USERETIONSHIP起作用，更改了模型关系，将日期key和到货日期key相连。

  • 3，计算最终模型（应用显式筛选器）

    将第一步计算的2007这个显式筛选器用于第二步的新模型，而内层的显式筛选器会覆盖中层的隐式筛选器，所以模型最终只有个显式筛选器：2007，2007沿着新的关系传递筛选，得到最终的模型可见数据，最后计算第一参数，得到的是2007年的到货量。

以上计值流计算3次，每次迭代，显式筛选器都是2007，因此3次迭代的计算结果都是2007年的到货量，所以“计值流实战”的结果是2007年到货量的3倍。
